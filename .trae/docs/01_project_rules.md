\# PropSys Frontend Rules



Proyecto objetivo: PropSys



Objetivo:

Construir solo el front-end de PropSys. No implementar backend real todavía.



Stack obligatorio:

\- Next.js

\- TypeScript

\- Tailwind CSS

\- shadcn/ui



Reglas:

\- No inventar backend real

\- Todo dato debe venir de mocks, adapters o interfaces

\- Código limpio, modular y reutilizable

\- Componentes compartidos primero

\- Responsive real

\- Accesibilidad básica

\- Estados de loading, empty, error y success

\- Route guards simulados por rol y estado

\- Mantener separación entre admin/staff y resident/owner



Páginas a construir:

\- /

\- /router

\- /setup

\- /admin/dashboard

\- /admin/receipts

\- /admin/receipts/\[id]

\- /resident/receipts

\- /resident/receipts/\[id]

\- /reset-password

\- /404



Prioridades:

1\. Base del proyecto

2\. Auth + router + setup

3\. Receipts admin y resident

4\. Dashboard admin

5\. 404 y polish visual



Criterios:

\- Corregir inconsistencias claras del sistema Bubble actual

\- Unificar navegación privada

\- Unificar headers

\- Unificar receipt rows

\- Unificar badges y empty states

\- Usar PropSys como nombre visible del producto
